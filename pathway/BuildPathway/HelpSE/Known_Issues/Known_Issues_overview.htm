OpenOffice/LibreOffice issues

Note

We want to have support for export to Microsoft Word® directly. Until then, consider the following:

  • OpenOffice/LibreOffice was implemented first because it is unencumbered software. That means our everyone can freely install the program without having to worry about licensing. It has roughly the same features as Word although the user interface is closer to Word 2003 than to Word 2007.

  • OpenOffice/LibreOffice can save the file in a Word format. So it is possible after the file appears in OpenOffice or LibreOffice to save it in Word format and then open it in Word.
    This is an extra step and a process that went directly to Word, could put the guide words in the header, and so on. Once the exported file appears in Word, you need to know how to insert the guide word in the header using the StyleRef field code.

    • The StyleRef field code needs to be used in Word to set up guide words for headers. This feature can not be automatically exported by OpenOffice/LibreOffice. Consult the Microsoft Word® help file for instructions on the user of this feature.
