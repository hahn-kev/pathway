Roles

In a typical language project, different roles exist naturally. Some individuals have more expertise at publishing (Global Publishing Services, Language Software Development, Focus group, and so on). Other individuals have more expertise regarding the language data. Various management structures typically exist.

Roles and some expected tasks are listed below. The goal is to effectively make the expertise of publishers available to language data experts.

Role 1 - Designers

  • Create standard stylesheets

  • Approve stylesheets

Role 2 - Manager

  • Set Defaults

  • Limit choices

  • Rename or edit stylesheets

  • Create custom stylesheets (New or Copy)

  • Add comments based on the use they envisage for the style sheet in the language project.

Role 3 - Printing Users

  • Create content (with a content-providing program)

  • Output the language data using the defaults (typical)

  • Manually adjust layout of output data (such as to reposition a picture in a lexical entry in LibreOffice)

  • Provide feedback to stylesheet designers and mangers
