Style names in the output file

When you export data, such as from SIL FieldWorks to LibreOffice, you see names of styles in LibreOffice that do not match those you see in Pathway Configuration Tool. Typically, you will see longer style names in the output file. This is especially true for character styles. During the export process, styles receive names based on which elements they contain.

Example

Some cascading stylesheet properties allow sizes to be set to relative values. A font size could be specified as 55%, which means 55% of the style tag that contains the element in question. It is important to know what element contains the current one. So, during the export process, styles are named in the resulting output with the names of the tags that contain this one.

  • entry_LetterData would mean the entry contained in the LetterData. This allows the style of a text element to vary based on the context in which it occurs.

  • xsensenumber is the style used for the number before the senses. It is the same style name whether the entry is a major entry, a minor entry, or a sub-entry. If you want to format these kinds of entries differently, having the subentry style named xsensenumber_sense_sensessub_subentry_subentries_entry_letData and the main entry style named xsensenumber_sense_senses_entry_letData could be helpful. If you wanted the main entries to be in 12pt and the subentries to be in 10pt., you could set the xsensenumber style to be 80% of the font size of the containing element.
