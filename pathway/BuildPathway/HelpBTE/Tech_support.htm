Error or Bug Reports, or Feature Requests

The Pathway Configuration Tool is under active development and is distributed as freeware. As such, no warranty or support is provided and you use it at your own risk.  We welcome error (crash) or bug reports, and feature requests.

The Issues tab at http://code.google.com/p/pathway/issues/list allows you to create and follow issues, which can be error or bug reports, or feature requests. Optionally, send them to pathway@sil.org.

Please include as much information as you can so the error or bug can be processed most efficiently:

  • How does the crash or problem occur? If at all possible, please include the necessary steps so we can reproduce the problem too.
    Where     does the crash occur (which feature was being used).
    When     does the crash occur? (Always/ Frequently/Sometimes/Whenever I ____. This is the first time).

  • Additional information can also be very useful:

    • Does this problem occur in every project or with every stylesheet, or only a particular one?

    • Which Operating System are you using?

      • Windows XP, Vista, Windows 7 or 8, or Linux?

      • Virtual machine?

      • 32 bit or 64 bit?

      • Which language version of the Operating System are you using, if it isn't English?

If you include a FieldWorks crash dump, the FieldWorks Version and the Operating System are included in that information. In this case, add How and When information.

Note

  • If you hold down the Shift key while you start Pathway Configuration Tool, settings files are changed back to the setting that appeared just after you installed the tool. If a problem you are experiencing is related to setting files, this can solve the problem. Be aware that your Set Defaults dialog box settings are cleared and you will need to set them again.
