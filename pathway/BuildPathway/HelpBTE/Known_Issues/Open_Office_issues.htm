OpenOffice/LibreOffice issues

In all cases, review the OpenOffice or LibreOffice Help topics for more up-to-date and complete information.

Complex fonts or Asian languages

OpenOffice and LibreOffice Writer may crash from data that needs complex rendering (is written with characters or glyphs that are composed of several parts), or is written from right-to-left. These are considered to be CTL (Complex Text Layout).

To prevent this crash, do the following in OpenOffice or LibreOffice Writer:

  • On the Tools menu, click Options. In the Options dialog box, expand Language Settings, and then click Languages.

    • Select () Enabled for complex text layout (CTL).

    • For Asian languages, also select Enabled for Asian languages.

    • Click OK.

Dashed borders

  • Dashed borders is not supported.

Large Files

  • Large files open and operate slowly.

    • You may want to select Split File by Letter (more) to see if that helps. You may want to combine the files into one larger master file later.

  • If you export to OpenOffice/LibreOffice, and if the output file is large (500 or more pages), or some of the content is formatted by many styles, then OpenOffice/LibreOffice can crash. In this case, you can select () the Replace styles with direct formatting in OpenOffice/LibreOffice check box in the Export Through Pathway or Set Defaults dialog box.

    • Be aware that when selected, the stylesheet is eliminated and style attributes are applied as direct formatting in OpenOffice/LibreOffice. Consequently, it will be inconvenient to change formatting after export.

  • http://help.libreoffice.org/Writer/Master_Documents_and_Subdocuments gives more information about large files.

Navigator (in OpenOffice/LibreOffice)

  • Press F5. It has a list of all the graphics in the file. It is helpful to select each one and see how it has been formatted. The Navigator is also very helpful when working with Master Documents.

Styles dialog (in OpenOffice/LibreOffice)

  • Press F11. The OpenOffice/LibreOffice Styles and Formatting dialog box has a tool bar with the different kinds of styles that are supported. The first icon on the tool bar is for paragraph styles. The second icon is for character styles. It is often useful to switch the list depending on whether the element in question is part of a paragraph or an entire paragraph.

Tables

  • Tables are not implemented yet in OpenOffice/LibreOffice export.

Word Spacing

  • "Word spacing" is not supported. When OpenOffice/LibreOffice is the target output, word spacing is 'greyed-out' (unavailable).

Vertical Alignment

  • Percentage oriented vertical alignment is not supported. When OpenOffice/LibreOffice is the selected destination, the valid property values for vertical alignment are restricted to those that OpenOffice/LibreOffice support.

Other

  • Security: OpenOffice/LibreOffice Macro Security must be reduced to medium (at most). After installing OpenOffice/LibreOffice, on the Tools menu click Options. In the Options dialog box, expand OpenOffice or LibreOffice and then click Security. Under Macro Security click the Macro Security button set the security level to Medium.

  • To run the macro, you can close and reopen the .odt file, or run the StartDontForget macro via the Tools menu.

  • Use Preview to force repagination. The page numbers change in the lower left corner of the window during the repagination.

  • Writing Systems (Headwords versus guide words): When you export a dictionary via Pathway, if the default vernacular writing system set in FieldWorks Language Explorer is not the writing system of the lexeme forms or citation forms, the guide words (headers) may use a different writing system than the dictionary headwords. Also, the sort order of the entries may be set by the wrong writing system.
